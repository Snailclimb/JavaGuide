## Rapid development platform

- [Snowy](https://gitee.com/xiaonuobase/snowy): China’s first national secret rapid development platform that separates front-end and back-end. Detailed introduction: [5.1k! This is the most powerful front-end and back-end separation rapid development scaffolding I have ever seen! ! ](https://mp.weixin.qq.com/s?__biz=Mzg2OTA0Njk0OA==&mid=2247534316&idx=1&sn=69938397674fc33ecda43c8c9d0a4039&chk sm=cea10927f9d68031bc862485c6be984ade5af233d4d871d498c38f22164a84314678c0c67cd7&token=1464380539&lang=zh_CN#rd).
- [eladmin](https://github.com/elunez/eladmin): A back-end management system that separates front-end and back-end. The project adopts module development method, permission control adopts RBAC, supports data dictionary and data permission management, supports one-click generation of front-end and back-end code, and supports dynamic routing.
- [RuoYi](https://gitee.com/y_project/RuoYi): RuoYi is a SpringBoot-based permission management system that is easy to read and understand, has a simple and beautiful interface, and can be used directly by running it.
- [AgileBoot-Back-End](https://github.com/valarchie/AgileBoot-Back-End): A basic rapid development framework based on Ruoyi that has done a lot of refactoring and optimization.
- [SmartAdmin](https://gitee.com/lab1024/smart-admin): A simple and easy-to-use low-code mid- and back-end solution.
- [EuBackend](https://gitee.com/zhaoeryu/eu-backend): A lightweight rapid development platform developed based on SpringBoot.
- [RuoYi-Vue-Pro](https://github.com/YunaiV/ruoyi-vue-pro): The new Pro version of RuoYi-Vue, optimized and reconstructed all functions, supports data permissions, SaaS multi-tenancy, Flowable workflow, three-party login, payment and other functions.
- [RuoYi-Vue-Plus](https://gitee.com/dromara/RuoYi-Vue-Plus): The new Plus version of RuoYi-Vue, which rewrites all the functions of RuoYi-Vue and integrates Sa-Token, Mybatis-Plus, Jackson, SpringDoc, Hutool, OSS regular synchronization, etc.
- [pig](https://gitee.com/log4j/pig "pig"): RBAC permission management system based on Spring Boot + Spring Cloud + OAuth2.
- [Guns](https://gitee.com/stylefeng/guns): A modern basic framework for Java application development.
- [JeecgBoot](https://github.com/zhangdaiscott/jeecg-boot): A J2EE low-code rapid development platform based on a code generator that supports the generation of projects with front-end and back-end separation architecture.
- [Erupt](https://gitee.com/erupt/erupt): A low-code full-stack class framework that uses Java annotations to dynamically generate pages and background functions such as add, delete, modify, check, and permission control.
- [BallCat](https://github.com/ballcat-projects/ballcat): A fully functional rapid development scaffolding! In addition to the most basic permission management and scheduled task functions, it also supports XSS filtering, SQL injection prevention, data desensitization and other functions.
- [JHipster](https://github.com/jhipster/generator-jhipster): Open source application platform to create Spring Boot + Angular/React projects in seconds.

## Wiki/Documentation System

- [zyplayer-doc](https://gitee.com/dromara/zyplayer-doc): A knowledge base, notes, and WIKI document management tool suitable for private deployment by teams and individuals. It also includes modules such as database management and API interface management.
- [kkFileView](https://gitee.com/kekingcn/file-online-preview): Document online preview solution, supports preview of almost all mainstream document formats, such as doc, docx, ppt, pptx, wps, xls, xlsx, zip, rar, ofd, xmind, bpmn, eml, epub, 3ds, dwg, psd, mp4, mp3, etc.

## File management system/network disk

- [cloud-drive](https://gitee.com/SnailClimb/cloud-drive): A minimalist modern cloud storage system based on Alibaba Cloud OSS, providing file upload, download, sharing and other functions. The system adopts a front-end and back-end separation architecture to provide safe and reliable file storage services.
- [qiwen-file](https://gitee.com/qiwen-cloud/qiwen-file): A distributed file system implemented based on SpringBoot+Vue. It supports multiple storage methods such as local disk, Alibaba Cloud OSS object storage, FastDFS storage, MinIO storage, etc. It supports office online editing, multi-part upload, technical second transfer, breakpoint resume transfer and other functions.
- [free-fs](https://gitee.com/dh_free/free-fs): A cloud storage management system based on SpringBoot + MyBatis Plus + MySQL + Sa-Token + Layui with Qiniu Cloud and Alibaba Cloud OSS. Includes functions such as file upload, deletion, online preview, cloud resource list query, download, file movement, renaming, directory management, login, registration, and permission control.
- [zfile](https://github.com/zfile-dev/zfile): An online network disk based on Spring Boot + Vue. It supports docking with S3, OneDrive, SharePoint, Google Drive, Duojiyun, Youpaiyun, local storage, FTP, SFTP and other storage sources. It supports online browsing of pictures, audio and video playback, text files, Office, obj (3d) and other file types.

## Mall system

Most of the mall systems below are relatively complex, such as mall. If you don’t have a Java foundation and are not familiar with Spring Boot, it is not recommended to over-study the following projects or use these projects as graduation projects.

- [congomall](https://gitee.com/nageoffer/congomall): A different TOC mall system, SpringCloud-Alibaba microservice architecture design, developed based on DDD domain-driven model, elegant code design, covering the core business of the mall.
- [mall](https://github.com/macrozheng/mall "mall"): The mall project is an e-commerce system, including a front-end mall system and a back-end management system, implemented based on SpringBoot+MyBatis.
- [mall-swarm](https://github.com/macrozheng/mall-swarm "mall-swarm"): mall-swarm is a microservice mall system that uses core technologies such as Spring Cloud Greenwich, Spring Boot 2, MyBatis, Docker, and Elasticsearch. It also provides a Vue-based management background to facilitate and quickly build the system.
- [litemall](https://github.com/linlinjava/litemall "litemall"): Another small mall. litemall = Spring Boot backend + Vue administrator frontend + WeChat applet user frontend + Vue user mobile terminal.
- [newbee-mall](https://github.com/newbee-ltd/newbee-mall): The newbee-mall project (Newbee Mall) is an e-commerce system, including newbee-mall mall system and newbee-mall-admin mall backend management system, developed based on Spring Boot 2.X and related technology stacks.

## Permission management system

Permission management systems are generally very important in enterprise-level projects. If you need to actually understand how a good permission system is designed, it is recommended that you refer to the following open source projects.

- [SpringBoot-Shiro-Vue](https://github.com/Heeexy/SpringBoot-Shiro-Vue): Based on the permission management idea of ​​Spring Boot-Shiro-Vue, both the front and back ends are controlled, and button/interface level permissions can be achieved.
- [renren-security](https://gitee.com/renrenio/renren-security): A flexible permission control system that can control pages or buttons to meet most permission requirements

## Make the wheel

- [guide-rpc-framework](https://github.com/Snailclimb/guide-rpc-framework): A custom RPC framework based on Netty+Kyro+Zookeeper - with detailed implementation process and related tutorials.
- [mini-spring](https://github.com/DerekYRC/mini-spring): A simplified version of the Spring framework that can help you quickly become familiar with the Spring source code and master the core principles of Spring. The code is extremely simplified and retains the core functions of Spring, such as IoC, AOP, resource loader and other core functions.
- [mini-spring-cloud](https://github.com/DerekYRC/mini-spring-cloud): A handwritten simplified version of Spring Cloud, designed to help you quickly become familiar with the Spring Cloud source code and master its core principles. Related reading: [Handwriting a simplified version of Spring Cloud! ](https://mp.weixin.qq.com/s/v3FUp-keswE2EhcTaLpSMQ).
- [haidnorJVM](https://github.com/FranzHaidnor/haidnorJVM): A simple version of Java virtual machine implemented in Java, introduction: <https://www.zhihu.com/question/28125278/answer/3137240457>.
- [itstack-demo-jvm](https://github.com/fuzhengwei/itstack-demo-jvm): Implement the basic functions of JVM through Java code (search and parse class files, bytecode commands, runtime data areas, etc.) Related reading: ["zachaxy's handwritten JVM series"](https://zachaxy.github.io/tags/JVM/).
- [Freedom](https://github.com/alchemystar/Freedom): DIY a database with ACID. Related projects: [MYDB](https://github.com/CN-GuoZiyang/MYDB) (a simple database implementation), [toyDB](https://github.com/erikgrinaker/toydb) (a distributed SQL database implemented in Rust).
- [lu-raft-kv](https://github.com/stateIs0/lu-raft-kv): A Java version of Raft(CP) KV distributed storage implementation, very suitable for research by friends who want to learn the Raft protocol in depth. lu-raft-kv has implemented two core functions of the Raft protocol: leader election and log replication. If you want to learn this project, it is recommended that you read the project introduction written by the author in advance, which is more detailed, at the address: <http://thinkinjava.cn/2019/01/12/2019/2019-01-12-lu-raft-kv/>.
